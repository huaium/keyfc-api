# keyfc-api overview
- Purpose: Kotlin Multiplatform library that parses the KeyFC web pages (index, forum, topic, login, search, user center, notifications, inbox, my topics/posts/favourites) into structured data classes so other Kotlin/JVM/Android/iOS/Linux clients can consume them.
- Tech stack: Kotlin (v2.2) Multiplatform with Gradle Kotlin DSL, targets for JVM, Android, iOS (X64/Arm64/Simulator), Linux X64, uses version catalog (`gradle/libs.versions.toml`), Ktor HTTP client (core, logging, platform engines), ksoup HTML parser, kotlinx-coroutines, kotlinx-datetime, Napier logging, SLF4J/Logback adapters, and Vanniktech Maven Publish for artifacts.
- Structure: root project contains two modules: `:library` (multiplatform sources under `library/src/{commonMain,androidMain,jvmMain,iosMain,linuxX64Main,...}` with subpackages `net.keyfc.api` plus `parser`, `model`, `ext`) and `:example` (simple JVM `kotlin("jvm")` module whose `src/main/kotlin` has subpackages for each example case that call `KeyfcClient`).
- Tests live under `library/src/commonTest/kotlin` but currently only the stub `ExampleTest` (all code is commented out), so running `./gradlew test` will succeed but there are no assertions yet.
- Entry points: each example package exposes a `fun main()` (e.g., `index.IndexExampleKt`) that can be executed by building the `:example` JAR (`./gradlew :example:build`) and running `java -cp example/build/libs/example.jar index.IndexExampleKt` (swap in the other packages such as `forum.ForumExampleKt`, `topic.TopicExampleKt`, etc.).
- Environment: development assumes standard Darwin shell utilities (`ls`, `cd`, `pwd`, `cat`, `sed`, `rg`, `find`) plus `git` for source control and the bundled Gradle wrapper (`./gradlew`).
